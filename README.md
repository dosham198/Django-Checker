# Django-Checker

Django로 개발된 웹 애플리케이션의 보안 취약점을 정적으로 탐지하는 도구입니다.
프로젝트 폴더 경로를 입력하면 `.py`, `.html` 파일을 탐색하고, 사전에 정의한 보안 규칙을 기준으로 취약 코드 패턴을 검사합니다.

## 프로젝트 개요
본 프로젝트는 보안에 취약한 웹 사이트를 직접 구현하고, 해당 취약점을 탐지할 수 있는 자동 분석 도구를 개발하는 것을 목표로 진행되었습니다.
Django-Checker는 Django 프로젝트의 소스코드를 대상으로 다음과 같은 취약점 패턴을 탐지합니다.

* SQL Injection
* XSS
* File Upload 취약점
* CSRF 관련 취약 설정
* DEBUG 모드 및 ALLOWED_HOSTS 설정 문제

## 주요 구조

```text
Django-Checker
├── main.py        # 프로그램 시작점
├── scanner.py     # 파일 탐색 및 코드 분석
├── rules.py       # 취약점 탐지 규칙 정의
├── reporter.py    # 탐지 결과 보고서 생성
└── reports/       # 분석 결과 저장 폴더
```

## 주요 기능

* 입력한 Django 프로젝트 경로의 `.py`, `.html` 파일 탐색
* 정규표현식을 기반으로 취약 코드 패턴 탐지
* 탐지된 취약점의 파일 경로, 라인 번호, 코드 내용 기록
* 분석 결과를 보고서 형태로 저장

## My Role

본 프로젝트에서 저는 `rules.py` 작성을 담당했습니다.

주요 역할은 다음과 같습니다.

* SQL Injection 탐지 규칙 작성
* XSS 탐지 규칙 작성
* File Upload 취약점 탐지 규칙 작성
* CSRF 관련 취약 설정 탐지 규칙 작성
* 주석 코드 오탐 방지를 위한 필터링 로직 설계
* 정상적인 외부 스크립트 사용과 위험한 인라인 스크립트를 구분하는 탐지 조건 작성

## 느낀 점

단순히 취약점을 이론으로 학습하는 것에서 그치지 않고, 취약한 웹 사이트를 직접 구현한 뒤 이를 탐지하는 도구까지 개발해보며 웹 취약점의 발생 원리와 정적 분석 방식의 한계를 함께 이해할 수 있었습니다.
